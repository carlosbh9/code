Logica en programacion - basico (HTML Y JavaScript)

Sean bienvenidos a este curso de lógica de programación con JavaScript y HTML. Si estás aquí es porque quieres aprender a programar y dar tus primeros pasos en este mundo de programación y de desarrollo de software.

Nosotros usamos la programación diariamente, en forma intrínseca, muchas veces hasta sin darnos cuenta. Por ejemplo cuando abrimos una aplicación en el celular, cuando llamamos a un Uber, esa aplicación está programada con programación, porque no hay una persona atrás cuando llamamos a un Uber que esté llamando al taxista, al chofer que nos va a recoger.

Cuando accesamos a una página web y hacemos una compra online, la misma cosa, ya la página web está programada para que cuando ejecutemos una compra, nos solicite todos nuestros datos y el producto que estamos solicitando vaya a una caja de pedidos, a un sector de pedidos.

También usamos programación a un cajero automático por ejemplo. Cuando vamos a sacar dinero no hay una persona atrás que nos esté entregando el dinero, ese cajero automático tiene una computadora que está programada para que nos pueda atender en forma automática en este caso.

Bueno, como podemos ver, la programación hace parte de nuestro día a día en este mundo digital. Para querer aprender a programar, el paso inicial es la lógica de programación, independientemente de cuál código, en cuál herramienta, es aprender la lógica de programación que aplica prácticamente para cualquier código y cualquier escenario.

Y bueno, lógicamente también necesitamos usar alguna herramienta que lea nuestro código. Nuestro browser, nuestro navegador es un claro ejemplo de ser un lector de código, aunque no lo parezca.

Aparte del navegador, que va a ser nuestro lector de código, necesitamos de editores de texto. El mismo, en el propio Windows, si son usuarios de Windows, van a tener Notepad, aquí por ejemplo, entonces pueden tener el Notepad, donde puedo editar texto. Pero este editor de texto particularmente no me gusta, no tiene muchas funcionalidades.

HTML5 Y CSS 3CSS

Hola amigos y amigas. Sean bienvenidos a la primera parte del curso HTML 5 y CSS 3. ¿Por qué HTML 5? Porque vamos a abordar en este curso la quinta versión de HTML, que es la que está actualmente en uso, y de la misma forma con el CSS. La tercera versión es la que se está usando en este momento.

El objetivo de este curso es introducirlos a este mundo amplio y fantástico de Front End, que Front End es aquella área o aquel equipo de personas que se encargan de la creación de páginas web y de la creación de la interfaz de muchos sistemas entre un usuario, entre el cliente, y el sistema.

Entonces, es aquella parte del sistema que el usuario ve y donde él coloca su nombre en un formulario o toda la interacción en sí que esa persona va a tener con una página web.

Entonces es un área superimportante, un área que está en constante crecimiento, porque cada día que pasa las empresas se están volviendo más digitales y es una necesidad hoy en día estar en la web, porque quien no está online prácticamente no existe. Entonces, en este proyecto vamos a abordar la construcción de una página web desde cero, desde lo más básico.

Vamos a empezar con nuestro código en blanco, básicamente, y vamos a ir aprendiendo de a poco a poco, paso a paso, cada uno de los elementos necesarios para construir una página web usando únicamente HTML y CSS.

Entonces, al término de este curso van a estar aptos para construir una página web básica para su empresa o para algún cliente que les solicite, ya sea una barbería, como es el caso de nuestro ejemplo que vamos a construir la Barbería Alura. Git y GitHub: controle y comparta su código

Hola a todos. Bienvenidos a este nuevo video de Git. Antes de empezar a usar Git, vamos a ver: ¿Para qué sirve y qué es un control de versiones? Imagina que sos un desarrollador, tenés proyectos y códigos en tu computadora y obviamente vas a ir alterando tu código. El problema viene cuando se trabaja en equipos.

Además de vos, hay otras personas que trabajan en ese proyecto. Cada uno tiene su propia copia del código y, como todos trabajamos en el mismo proyecto, cada uno de nosotros va a hacer modificaciones y tenemos que notificar a las otras personas que hicimos modificaciones. El problema es que cuando enviamos el código por e-mail, pendrive, Dropbox, etcétera. ¿Qué pasa si ellos ya hubieran modificado el código?

¿Cómo sabés si primero hay que agarrar las versiones de ellos y luego tenemos que colocar nuestro código encima? ¿Qué parte deberíamos modificar con nuestro código y qué parte deberían quedar en el código de nuestros compañeros? Eso es un problema. Para esto, una posible solución es colocar un servidor específico para enviar nuestras alteraciones, entonces cuando finalizamos una modificación, la enviamos a ese servidor.

Y si otra persona quiere mandar una modificación de su código, necesitamos una herramienta que gestione el código y diga: “Esa versión en la que basaste tu código no es la más actualizada. Hay una versión nueva”. Y así, no deja que se envíe directamente ese archivo. Lo que pide es que antes de enviar las modificaciones, esa persona tiene que bajar la versión más nueva, para que ahí pueda modificar solo la parte de código que tuvo cambios.

Y luego sí, pueda enviar ese código al servidor. A eso es a lo que se llama control de versión. Si tenemos diferentes versiones del código, necesitamos un sistema que controle esas versiones y para eso sirve Git.
